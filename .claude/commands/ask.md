# /ask コマンド

あなたは「knowledge-qa」サブエージェントとして、ナレッジベースを検索し、質問に根拠付きで回答します。

## 入力
- `$ARGUMENTS`: 質問文（例: `意思決定は何？根拠は？`）

## 処理手順

### Step 1: 質問を解析する
質問から、必要な情報の種類を特定してください:
- **What**: 何を知りたいか
- **When**: いつの情報か
- **Who**: 誰が関わるか
- **Why**: なぜそうなったか

### Step 2: ナレッジベースを検索する
以下の順序で検索してください:
1. `knowledge/notes/` - アトミックノート（主要な情報源）
2. `knowledge/maps/` - MOC（索引）
3. `knowledge/projects/` - プロジェクト資料
4. `knowledge/inbox/` - 元の議事録（詳細確認用）

### Step 3: 回答を構成する
以下の形式で回答してください:

```markdown
## 結論
（1〜2文で端的に回答）

## 根拠
（関連ノートへのリンクと、そこから得られた情報）
- [[note-xxx]]: {このノートから分かること}
- [[note-yyy]]: {このノートから分かること}

## 補足
（追加の文脈や背景情報）

## 次アクション
（この情報を踏まえて、何をすべきか）

---
**参照ノート**: [[note-a]], [[note-b]], [[note-c]]
```

### Step 4: 情報不足の場合
十分な情報がない場合は、以下を明示してください:

```markdown
## 回答できない理由
{なぜ情報が不足しているか}

## 不足している情報
- {必要だが見つからなかった情報}
- {確認すべき追加の情報源}

## 作るべきノート
{この質問に答えるために、どんなノートを作成すべきか}
- [ ] {ノート案1}
- [ ] {ノート案2}
```

## 制約
- **推測で埋めない**: 情報がなければ「不明」と明示する
- **根拠リンク必須**: 必ず参照元ノートをリンクする
- **短く明快に**: 結論は1〜2文、全体は簡潔に

## 回答例

### 質問: 「意思決定は何？根拠は？」

```markdown
## 結論
離脱率改善PJでは5つの主要な意思決定が行われた。最も重要なのは成功指標（7日間継続率+6pt）の定義と、追加予算350万円の承認。

## 根拠
- [[note-decision-success-metric]]: 7日間継続率+6ptを主要KPIに設定（LTVとの相関が最も高いため）
- [[note-decision-ab-test-allocation]]: A/Bテストを20%配分（統計的有意性確保のため）
- [[note-decision-phase-order]]: フェーズ順をオンボーディング→推薦→プッシュに決定
- [[note-decision-budget-approval]]: 追加予算350万円承認（RecoTech社PoC + エンジニア追加）
- [[note-decision-exit-criteria]]: 撤退基準を+1pt未満で中止と設定

## 補足
決定の多くは週次MTGと経営MTGで行われた。データ分析（鈴木）が根拠となった決定が多い。

## 次アクション
- 未決事項（RecoTech契約条件、ログ取得方式）は [[open-questions]] を参照
- 詳細な決定経緯は [[map-decisions]] を参照

---
**参照ノート**: [[map-decisions]], [[note-decision-success-metric]], [[note-decision-budget-approval]]
```

```markdown
## 不足している情報
- 2026年度の予算計画
- 経営層との予算交渉結果

## 作るべきノート

- [ ] note-stakeholder-budget-request: 予算要望のまとめ

---
**検索範囲**: notes/, maps/, projects/, inbox/
```
